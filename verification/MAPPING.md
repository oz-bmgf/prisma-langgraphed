## Orphans — In-scope old behavior with no LangGraph equivalent

Behaviors present in the old repo's analyze pipeline that have no clear counterpart in the new implementation.

1. **`document_catalog.json`-first fallback in `load_collection`** — old code tries `document_catalog.json` before falling back to `doc_list.json`. New `load_collection` node reads only `doc_list.json`. Collections that ship only `document_catalog.json` will fail silently.

2. **`_resolve_pages_dir` fallback to `etl/`** — old code falls back from `pages/` to `etl/` directory layout. New `load_collection` node sets `pages_dir` to a fixed path; the `etl/` fallback may be absent, breaking collections with the older layout.

3. **`build_asset_registry` + `write_run_assets` → `{output_dir}/assets.json`** — old `_cmd_analyze` called these explicitly when `--aux-collections` was set. New design routes aux backends via `factory.py` but the explicit asset-registry build and `assets.json` file write has no identified equivalent node or state field.

4. ~~**Sub-stage 3.1a — BOW-level causal model extraction (`extract_bow_causal_models`)**~~ — **RESOLVED 2026-05-29**: `extract_bow_causal_models` node added to causal subgraph; `_BOW_CAUSAL_PROMPT`/`_BOW_CAUSAL_SYSTEM` ported from OLD; `bow_causal_models: dict` added to `CausalState`; topology now 22 nodes.

5. ~~**Sub-stage 3.1b — investment link mapping / framing (`frame_investment_links`)**~~ — **RESOLVED 2026-05-29**: `frame_investment_links` node added; Case A tags existing links with BOW names (word-overlap); Case B creates framed links for zero-link investments.

6. ~~**Sub-stage 3.4b — orphan BOW link investigation (`investigate_orphan_links`)**~~ — **RESOLVED 2026-05-29**: `investigate_orphan_links` node added; runs after `collect_link_assessments`; for each unfunded BOW link: `search_portfolio` + `search_web` + LLM criticality assessment; appends to `link_assessments`.

7. ~~**`claim_investigator.py` tool set: `search_bow`, `search_science`, `search_policy`, `search_all`**~~ — **RESOLVED 2026-05-29**: `search_bow`, `search_science`, `search_policy` added as `@tool` functions in `investigation_tools.py`; added to `INVESTIGATION_TOOLS`, `_SUPPORTED_TOOLS`, `_COLLECTION_TOOLS`, and `_execute_actions` dispatch branches; `InvestigationAction.tool` schema and `INVESTIGATION_TOOL_DESCRIPTIONS` updated. `search_all` behaviour preserved via `search_portfolio` dispatch (no collection filter).

8. ~~**`science_investigator.py` own 8-tool set: `search_bow`, `search_science`, `search_policy`, `search_all`, `read_pages`**~~  — **RESOLVED 2026-05-29**: `SCIENCE_TOOLS` rebuilt to `[search_asta, search_bow, search_science, search_policy, search_web, read_document, compute, read_section]`; `submit_findings` and document-nav tools removed; `ScienceAction.tool` schema and `SCIENCE_INVESTIGATE_SYSTEM` prompt updated. `science_tools.py` module docstring updated to reflect the correct 8-tool set and rationale for exclusions.

9. **`exec_summary/premise_investigator` — 3-pass per-bet scientific premise cascade** — old exec summary ran an agentic 3-pass investigation (Investigation → Key Findings → Exec Summary) with OpenTelemetry spans and ASTA calls via `AstaClient`. New `report_assembler.py` (511 lines) handles executive summary generation but the full agentic 3-pass cascade with ASTA is not clearly present.

10. **`investigation_loop.py` — OpenAI Responses API with `previous_response_id` (stateful conversation)** — old loop used the Responses API to maintain a stateful conversation thread across iterations. New `core/investigation.py` appears to use `acall_llm` with stateless chat completions. If confirmed, the investigation strategy differs materially (stateless vs stateful).

11. **`CollectionTools.rerank(chunks, query, top_k)`** — explicit cross-strategy reranking method on the tools object. New rubric evaluator uses `_EmbeddingIndexAdapter` bridge to call backend methods; the `rerank` method itself has no direct counterpart in `collection_tools.py`.

12. **`FilterSpec` frozen dataclass** — old `search/base.py` defined `FilterSpec` with `supports_bow_id` flag. New `backends/base.py` has no `FilterSpec`; filter parameters are passed inline to `SearchBackend.search()`.

13. **`hybrid_search()` as a protocol-level method** — old `SearchIndex` protocol included `hybrid_search`. 15. **`llm_utils.trace_llm_context()` ContextVar context manager** — old LLM infrastructure used this for per-call trace scoping. New code uses OTEL/LangSmith auto-instrumentation; `_TRACE_CTX` ContextVar does not appear in new `llm_utils.py`.

## Extras — LangGraph nodes/tools in the analyze pipeline with no basis in the old repo

Constructs present in the new implementation that have no direct equivalent in the old-repo analyze pipeline.

1. **`load_catalog` node (Phase 0)** — no-op when data is already in state; enables the analyze subgraph to run standalone without the top-level graph. Old code always required `load_collection()` to run first as a separate function call before `run_research_analyst`.

2. **Two-level narrative fan-out (Phase 2.6)** — new analyze subgraph has `dispatch_investment_narratives` → `generate_investment_narrative` → `collect_investment_narratives` → `dispatch_scope_syntheses` → `generate_scope_synthesis` → `collect_timeline_narratives`. Old code had a single-level per-scope narrative call. The per-investment intermediate level is new.

3. **`_EmbeddingIndexAdapter` and `_SearchBackendToolsBridge` in `causal.py`** — adapter classes that bridge the new `SearchBackend` protocol to the duck-typed `tools` interface expected by `core/rubric_evaluator.py`, `core/investigation.py`, etc. Pure LangGraph porting artifact with no old-repo equivalent.

4. **`_classify_scope_fit` in `causal.py`** — deterministic scope-fit classification using term-list matching (no LLM). No equivalent in old `causal_pipeline.py`.

5. **`merge_scope_outputs` reducer** — custom deep-merge by `scope_id` for `Send()` fan-out accumulation. Old code mutated `ScopeOutput` objects directly in place; LangGraph's immutable state model requires this reducer.

6. **`_take_update` reducer** — last-writer-wins reducer for `WorkflowState` fan-out fields. No old-repo equivalent.

7. **`timeline_narrative_results` as a `Annotated[list, operator.add]` set-membership field** — used for idempotency checking instead of the old disk-based SHA256 cache. Enables resume without re-running cached narratives via LangGraph checkpoint state.

8. **`ScopeNarrativeState` and `ScopeSynthesisState` sub-state slices** — new TypedDicts for the two-level narrative fan-out. No old equivalents.

9. **Phase 3.5 (analyze subgraph) — `dispatch_investment_reports` → `build_investment_report_worker`** — blind AI-vs-team verdict synthesis as a separate Send() fan-out after the causal pipeline. Old code did AI-vs-team verdict inline within the causal pipeline synthesis steps (not as a separate parallel fan-out in the outer subgraph).

10. **Phase 3.6 (analyze subgraph) — `dispatch_scope_sections` → `synthesize_scope_section_worker`** — section-draft synthesis as a separate Send() fan-out in the analyze subgraph. Old code generated section drafts within the causal pipeline's per-scope BOW synthesis step (sub-stage 3.6), not as a post-causal fan-out.

11. **`interrupt_before` / `review_research_plan` / `approve_report` human-in-the-loop nodes** — human approval gates wired into the graph topology. Old pipeline had no in-process human interrupts; humans would run the next CLI command manually.

12. **`WorkflowState.research_model` and `WorkflowState.synthesis_model` as checkpointed state fields** — old code passed model names as function arguments, not checkpointed state. New design checkpoints model selection so resumed runs use the same models as the original invocation.

13. **`acall_structured(prompt, system, *, model, schema)` function in `llm_utils.py`** — Pydantic-validated LLM call returning a typed model instance. Old `call_llm(json_mode=True)` returned raw dict/str and used `safe_parse_json` for extraction.

14. **`AsyncSqliteSaver` / `AsyncPostgresSaver` checkpointer** — LangGraph persistence layer replacing old per-scope JSON checkpoint files. Enables full graph-level resume from any node boundary.

15. **`SPLIT_CHUNKS=12K` and `SPLIT_INVS=8` scope-splitting thresholds in `compute_scopes`** — new scope-splitting logic that breaks large BOWs into sub-scopes. Old `_compute_thread_scopes` had chunk-count gates but may not have had explicit split thresholds.

16. **`acall_llm` + LangChain `ChatAnthropic`/`ChatOpenAI` as the unified LLM gateway** — old code routed through `call_llm` which called `_call_anthropic`/`_call_openai_fallback` in a separate `synthesis` package. New code is fully LangChain-native and auto-instrumented by `LangChainInstrumentor`.

17. **`EdisonAgentState`, `GsVerifierState`, `EvidenceAuditState`, `FindingVerificationState` in `state.py`** — TypedDicts for out-of-scope graphs defined in the same state module. No analyze-pipeline impact but present in the implementation.
